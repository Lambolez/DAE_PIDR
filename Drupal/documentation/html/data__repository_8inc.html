Detailed Description

Functions handling raw SQL interaction with the backend data base through Drupal.

Author:
Bart Lamiroy lamir.nosp@m.oy@c.nosp@m.se.le.nosp@m.high.nosp@m..edu
Date:
May 2010
Version:
1.0

This file contains all necessary code to implement the Data Repository Module which allows authorized Drupal users to interact directly with the backend data base management system using raw, unfiltered SQL.

Users need to have the Drupal 'raw DAE database access' access rights set.

data_repository_query ( )

Drupal menu callback for Repository as declared in data_repository_menu()

Creates a $_SESSION variable for memorizing cross-page query inputs.

Displays the form defined in data_repository_sql().

Drupal automagically attaches data_repository_sql_submit() as the form submit callback.

data_repository_sql ( $  form_state)

Form definition function for SQL query input.

The form contains only two fields:

  • a textarea field named 'query'
  • a submit button named 'submit'
Parameters:
[in]$form_state
Returns:
a Drupal form array.

Checks whether there is a previously known form entry for the query. If true, then pre-fill the form with this query. If not pre-fill the form with a default query.
